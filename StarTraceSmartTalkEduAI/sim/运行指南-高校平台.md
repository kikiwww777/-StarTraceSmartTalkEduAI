# 高校平台运行指南

## 快速开始

### 方式一：使用 Docker Compose（推荐）

这是最简单的方式，适合快速启动和测试。

#### 前置要求
1. **安装 Docker Desktop**
   - Windows: 下载并安装 [Docker Desktop for Windows](https://www.docker.com/products/docker-desktop/)
   - 确保 Docker Desktop 正在运行（系统托盘图标显示为运行状态）

#### 启动步骤

1. **打开终端/命令行，进入项目目录**
   ```bash
   cd "D:\code\Sim Studio\Sim Studio\sim"
   ```

2. **启动所有服务**
   ```bash
   docker compose -f docker-compose.prod.yml up -d
   ```

3. **等待服务启动完成**
   - 首次启动会下载 Docker 镜像，可能需要几分钟
   - 等待所有容器状态显示为 "healthy"

4. **访问应用**
   - 打开浏览器访问：**http://localhost:3000**
   - 会自动跳转到高校平台主页面

#### 停止服务
```bash
docker compose -f docker-compose.prod.yml down
```

#### 查看日志
```bash
docker compose -f docker-compose.prod.yml logs -f
```

### 方式二：使用 NPM 包（最简单）

如果你只想快速体验，可以使用官方提供的 NPM 包：

```bash
npx simstudio
```

这会自动启动 Docker 容器并运行应用，访问 http://localhost:3000

### 方式三：手动开发模式（需要配置数据库）

适合需要修改代码的开发场景。

#### 前置要求
1. **安装 Bun** (>=1.2.13)
   - 访问 https://bun.sh 安装
   - 或使用：`npm install -g bun`

2. **安装 Node.js** (>=20.0.0)
   - 访问 https://nodejs.org 下载安装

3. **安装 PostgreSQL** (12+)
   - 需要支持 pgvector 扩展
   - 推荐使用 Docker 运行：
   ```bash
   docker run --name simstudio-db \
     -e POSTGRES_PASSWORD=your_password \
     -e POSTGRES_DB=simstudio \
     -p 5432:5432 -d \
     pgvector/pgvector:pg17
   ```

#### 启动步骤

1. **安装依赖**
   ```bash
   cd "D:\code\Sim Studio\Sim Studio\sim"
   bun install
   ```

2. **配置环境变量**
   ```bash
   cd apps/sim
   cp .env.example .env
   ```
   
   编辑 `.env` 文件，设置必要的环境变量：
   ```bash
   DATABASE_URL="postgresql://postgres:your_password@localhost:5432/simstudio"
   BETTER_AUTH_SECRET="your_secret_key"  # 运行: openssl rand -hex 32
   BETTER_AUTH_URL="http://localhost:3000"
   NEXT_PUBLIC_APP_URL="http://localhost:3000"
   ENCRYPTION_KEY="your_encryption_key"  # 运行: openssl rand -hex 32
   ```

3. **配置数据库包环境**
   ```bash
   cd packages/db
   cp .env.example .env
   ```
   
   编辑 `packages/db/.env` 文件：
   ```bash
   DATABASE_URL="postgresql://postgres:your_password@localhost:5432/simstudio"
   ```

4. **运行数据库迁移**
   ```bash
   cd packages/db
   bunx drizzle-kit migrate --config=./drizzle.config.ts
   ```

5. **启动开发服务器**
   
   从项目根目录运行（推荐，同时启动应用和实时服务）：
   ```bash
   bun run dev:full
   ```
   
   或者分别启动：
   
   **终端1 - Next.js 应用：**
   ```bash
   bun run dev
   ```
   
   **终端2 - 实时 Socket 服务：**
   ```bash
   cd apps/sim
   bun run dev:sockets
   ```

6. **访问应用**
   - 打开浏览器访问：**http://localhost:3000**
   - 会自动跳转到高校平台主页面

## 使用说明

### 高校平台功能

访问 http://localhost:3000 后，会自动跳转到高校平台主页面，包含以下功能：

1. **登录/注册**
   - 点击右上角"登录"按钮
   - 使用邮箱注册或登录

2. **选择角色**
   - 登录后需要选择身份：**教师** 或 **学生**
   - 选择后即可访问对应的工作区

3. **功能模块**
   - **课程管理**：创建和管理课程
   - **班级管理**：创建和管理班级
   - **智能体操作**：创建和使用AI智能体
   - **教师工作区**：教师专属功能（仅教师可见）
   - **学生工作区**：学生专属功能（仅学生可见）

## 常见问题

### 1. 端口被占用
如果 3000 端口被占用，可以修改端口：
- Docker: 修改 `docker-compose.prod.yml` 中的端口映射
- 开发模式: 修改 `apps/sim/package.json` 中的 `dev` 脚本

### 2. 数据库连接失败
- 确保 PostgreSQL 正在运行
- 检查 `.env` 文件中的 `DATABASE_URL` 是否正确
- 确保数据库已创建并运行了迁移

### 3. Docker 容器无法启动
- 确保 Docker Desktop 正在运行
- 检查端口是否被占用
- 查看容器日志：`docker compose -f docker-compose.prod.yml logs`

### 4. 页面无法访问
- 确保服务已完全启动（等待容器健康检查通过）
- 检查防火墙设置
- 尝试清除浏览器缓存
