星溯智语EduAI智能体教育平台

执行概要
本文档为一套基于开源系统Sim Studio(https://github.com/simstudioai/sim)的二次开发设计方案，目标是构建面向高等教育的AI智能体工作台。通过充分利用Sim的核心能力，用最短的开发周期在Phase 1完成一个即插即用、立即可用的教学平台，后续Phase 2和3用于功能扩展和深度集成。
核心策略: - 不修改Sim核心代码 - 基于Sim的标准扩展机制和API - 优先构建教学层 - 模板库、课程管理、权限分离 - 充分复用基础设施 - 工作流编排、模型调用、MCP工具、向量数据库 - 快速迭代 - Phase 1聚焦必要功能，后续持续演进

第二部分：Phase 1 开发方案（最小可用系统）
2.1 Phase 1 目标
目标: 在2-3个月内交付一个教学可用的系统，支持以下核心场景:
1.	教师创建课程并发布模板式工作流
2.	学生基于模板构建和运行智能体
3.	系统追踪成本与配额，防止成本超支
4.	支持多个LLM提供商的灵活选择
5.	提供知识库上传与检索能力
不包括（后续Phase）: - 多智能体间的复杂协作框架 - 自然语言建模（自动生成工作流） - 高级评估与优化建议 - 深度LMS集成
2.2 Phase 1 核心模块
模块A: 课程与权限管理
功能: - 教师创建课程，学生注册加入 - 角色系统：管理员、教师、学生 - 课程级权限：允许使用的模型、工具、外部API - 工作区隔离：每个课程一个独立的工作空间
实现方式: - 在Sim的现有工作区（workspace）概念基础上扩展 - 添加课程表、学生名单等关系表 - 权限中间件拦截Sim API调用，检查用户权限
输出: - 后端API: /api/course/* (创建、列表、成员管理) - 前端：课程选择器、班级成员列表
模块B: 教学模板库
11个教育模板的参考实现（基于Sim的标准工作流格式）:
1.	文献综述生成 - 输入主题 → 自动爬虫+LLM → 生成综述
2.	课程学习助手 - 输入知识点 → 检索课程资料 → 讲解+例题
3.	知识图谱可视化 - 输入内容 → 抽取关键概念 → 生成图谱
4.	课件自动生成 - 输入大纲 → 分段讲义 → PPT脚本
5.	作业自动批改 - 输入学生作业 → 内容评分 → 生成反馈
6.	学术趋势分析 - 检索文献 → 趋势抽取 → 预测方向
7.	学习路径推荐 - 用户档案 → 推荐课程 → 进度计划
8.	协同写作与讨论 - 多人工作流 → 写作 → 实时反馈
9.	抄袭检测 - 输入文本 → 相似度对比 → 报告
10.	实验设计辅助 - 学科需求 → 实验框架 → 数据分析工具
11.	学术写作优化 - 论文输入 → 语法和风格检查 → 改进建议
实现方式: - 每个模板是一个导出的Sim工作流JSON文件 - 包含完整的节点配置、连线、提示词 - 教师可通过”导入模板”一键加载 - 学生基于模板创建个人副本，修改参数后运行
特点: - 模板自带使用文档（嵌入为节点的备注） - 模板自带改造指南（推荐修改点标注） - 支持版本管理（允许教师发布改进的版本）
存储形式:
templates/

├── 02_course_tutor.json
├── 03_knowledge_graph.json
├── 04_auto_ppt.json
├── 05_homework_grading.json
├── 06_trend_analysis.json
├── 07_learning_path.json
├── 08_collaborative_writing.json
├── 09_plagiarism_check.json
├── 10_experiment_design.json
├── 11_writing_optimization.json
└── README.md (模板目录说明)
模块C: 成本与配额管理
功能: - 为每个学生设置每日/每周Token配额 - 实时监控每个工作流执行的成本 - 超配额时警告或限制 - 教师可查看全班总成本与分布
实现方式: - 在Sim的执行日志表中添加cost_tokens字段 - 新增quota表：记录课程级和学生级配额 - 执行前检查：若预估成本超配额，弹出确认或阻止 - 后端中间件拦截LLM调用，计算并累加成本
UI集成: - 工作流执行页面：显示”预估成本: 5000 tokens / 剩余配额: 50000” - 学生仪表板：每日成本柱状图、剩余配额进度条 - 教师报告：班级成本TOP 10学生、课程总消耗
模块D: 知识库管理
功能: - 教师上传课程讲义、教材、论文等资料 - 系统自动分段、嵌入、索引到向量数据库 - 学生在工作流中使用”知识检索”节点查询相关资料 - 支持多个独立的知识库（按课程/专题隔离）
实现方式: - 复用Sim的pgvector集成（已内置） - 新增knowledge_base和knowledge_doc表 - 文件上传处理：PDF/Word/Markdown → 分段 → embedding → 存储 - 提供/api/knowledge-base/search端点供工作流节点调用
存储策略: - 向量表：knowledge_embeddings (doc_id, embedding, content_chunk) - 元数据表：knowledge_docs (id, kb_id, filename, upload_date, creator_id)
模块E: 前端UI增强
增强内容（不修改Sim核心编辑器）:
1.	左侧边栏新增：
o	“课程选择”下拉框
o	“模板库”快速访问
o	“我的配额”实时显示
2.	新增页面：
o	/courses - 课程列表与管理
o	/templates - 模板库浏览与导入
o	/dashboard - 学生学习成果板

3.	工作流执行页面增强：

2.3 Phase 1 数据库设计（新增表）
-- 课程管理
CREATE TABLE courses (
  id UUID PRIMARY KEY,
  name VARCHAR(255),
  description TEXT,
  instructor_id UUID REFERENCES users(id),
  workspace_id UUID REFERENCES workspaces(id),  -- Sim原有
  created_at TIMESTAMP,
  updated_at TIMESTAMP
);

CREATE TABLE course_members (
  id UUID PRIMARY KEY,
  course_id UUID REFERENCES courses(id),
  user_id UUID REFERENCES users(id),
  role ENUM('teacher', 'student'),  -- Sim有users，扩展角色
  created_at TIMESTAMP
);

-- 配额与成本
CREATE TABLE quota_settings (
  id UUID PRIMARY KEY,
  course_id UUID REFERENCES courses(id),
  token_limit_daily INT,
  token_limit_weekly INT,
  created_at TIMESTAMP
);

CREATE TABLE user_quota_usage (
  id UUID PRIMARY KEY,
  course_id UUID REFERENCES courses(id),
  user_id UUID REFERENCES users(id),
  date DATE,
  tokens_used INT,
  workflow_count INT,
  updated_at TIMESTAMP
);

CREATE TABLE execution_costs (
  id UUID PRIMARY KEY,
  workflow_id UUID REFERENCES workflows(id),  -- Sim原有
  execution_id UUID,
  user_id UUID REFERENCES users(id),
  input_tokens INT,
  output_tokens INT,
  total_tokens INT,
  cost_usd DECIMAL(10, 4),
  model_used VARCHAR(255),
  created_at TIMESTAMP
);

-- 知识库
CREATE TABLE knowledge_bases (
  id UUID PRIMARY KEY,
  course_id UUID REFERENCES courses(id),
  name VARCHAR(255),
  description TEXT,
  created_by UUID REFERENCES users(id),
  created_at TIMESTAMP
);

CREATE TABLE knowledge_documents (
  id UUID PRIMARY KEY,
  kb_id UUID REFERENCES knowledge_bases(id),
  filename VARCHAR(255),
  content_type VARCHAR(50),
  file_path VARCHAR(512),
  total_chunks INT,
  created_by UUID REFERENCES users(id),
  created_at TIMESTAMP
);

CREATE TABLE knowledge_embeddings (
  id UUID PRIMARY KEY,
  doc_id UUID REFERENCES knowledge_documents(id),
  chunk_index INT,
  content TEXT,
  embedding vector(1536),  -- pgvector支持
  metadata JSONB
);

-- 模板管理
CREATE TABLE templates (
  id UUID PRIMARY KEY,
  name VARCHAR(255),
  description TEXT,
  category ENUM('tutor', 'grading', 'writing', ...),
  workflow_json JSONB,  -- 完整的Sim工作流定义
  version INT,
  created_by UUID REFERENCES users(id),
  created_at TIMESTAMP
);

CREATE TABLE template_usage (
  id UUID PRIMARY KEY,
  template_id UUID REFERENCES templates(id),
  user_id UUID REFERENCES users(id),
  course_id UUID REFERENCES courses(id),
  used_at TIMESTAMP
);

{

  "tags": ["教学", "学习辅导", "RAG"],
  "difficulty": "beginner",
  "estimatedCost": "~5000 tokens",
  "nodes": [ /* Sim节点定义 */ ],

  "documentation": {
    "usage": "...",
    "keyPoints": [...],

  }
}
3.2 11个模板详细规格
模板1: 课程学习助手 ⭐ (重点模块)
场景: 学生输入知识点名称 → 系统讲解 + 提供例题 + 互动提问
工作流节点结构:
[输入: 知识点名称]
    ↓
[检索课程知识库 (RAG)]
    ↓
[生成讲解 (LLM: gpt-4)]
    ↓
[生成例题 (LLM: gpt-3.5-turbo, 成本优化)]
    ↓
[输出: 讲解 + 例题]
关键配置: - 知识库检索：top-k=5，相似度阈值=0.7 - LLM提示词：包含Socratic方法论，鼓励学生思考 - 支持多轮对话（学生可继续追问） - 成本估算：~3000-5000 tokens/次
预设改造方向: - 添加”难度调整”（简单/中等/困难） - 添加”语言切换”（中英文） - 集成”评分”（学生对讲解的满意度）

模板2: 文献综述生成
场景: 输入研究主题 → 搜索相关文献 → 生成综述文章
工作流节点结构:
[输入: 研究主题]
    ↓
[网页搜索 (Google Search / Perplexity)]
    ↓
[并行处理多篇文献]
    ├─ [LLM提取摘要与关键点]
    ├─ [LLM提取主要发现]
    └─ [LLM提取研究空白]
    ↓
[汇聚: 综合分析]
    ↓
[LLM生成结构化综述]
    ↓
[输出: Markdown格式综述]
关键配置: - 搜索数量：默认10篇，可调整 - 文献处理：支持PDF解析（通过MCP） - LLM模型：使用更强的模型（claude-3或gpt-4） - 输出格式：背景、发展阶段、主要成果、研究空白、未来方向
成本: - 搜索：~5000 tokens - 总体：~20000-30000 tokens

模板5: 智能作业批改与反馈
场景: 学生提交作业（论文/代码/问答）→ 自动评分与个性化反馈
工作流节点结构:
[输入: 学生作业 + 评分标准]
    ↓
[作业类型识别]
    ├─ [论文批改流程]
    │  ├─ 结构检查
    │  ├─ 内容分析
    │  ├─ 语言质量
    │  └─ 学术规范

    │  ├─ 语法检查
    │  ├─ 逻辑分析
    │  └─ 性能评估
    └─ [问答评分流程]
       ├─ 完整性检查
       ├─ 准确性判断
       └─ 深度分析
    ↓
[综合评分 (LLM)]
    ↓
[生成个性化反馈建议]
    ↓
[输出: 评分 + 反馈报告 + 改进建议]

成本: - 短作业（<500词）：~2000 tokens - 长论文（2000-5000词）：~5000-10000 tokens

成本: - 文献检索：~10000-20000 tokens - 总体：~25000-40000 tokens

4.2 工作流执行的成本计算
伪代码:
def calculate_execution_cost(workflow_execution):
    total_cost = 0
    
    for node in workflow_execution.nodes:
        if node.type == "llm":
            # 从LLM调用日志获取token数
            input_tokens = node.metadata.input_tokens
            output_tokens = node.metadata.output_tokens
            model = node.config.model
            
            # 查询模型的token价格表
            price = get_model_price(model)
            node_cost = (input_tokens * price.input + 
                        output_tokens * price.output)
            total_cost += node_cost
            
        elif node.type == "api_call":
            # API调用根据类型计费（如搜索、检索）
            if node.config.service == "web_search":
                total_cost += 0.05  # 每次搜索$0.05
            elif node.config.service == "embedding":
                total_cost += (node.metadata.vector_count * 0.0001)
    
    return total_cost

4.3 知识库的检索与增强
RAG流程:
1. 用户查询: "什么是机器学习？"
2. 生成embedding: query_emb = embed("什么是机器学习？")
3. 向量搜索: results = search_similar(query_emb, kb_embeddings, top_k=5)
4. 拼装上下文: context = "\n".join([r.content for r in results])
5. LLM回答: response = llm(prompt + context + query)
Sim中的集成: - 在工作流中添加”知识库检索”节点 - 节点配置: 知识库ID、top-k、相似度阈值 - 输出: 相关文段列表 (id, content, similarity_score)
4.4 权限与多租户隔离
权限检查中间件:
// Express中间件示例
async function checkCoursePermission(req, res, next) {
  const courseId = req.params.courseId;
  const userId = req.user.id;
  
  const member = await db.course_members.findOne({
    course_id: courseId,
    user_id: userId
  });
  
  if (!member) {
    return res.status(403).json({ error: "Access denied" });
  }
  
  // 根据角色设置权限
  if (member.role === 'student') {
    req.permissions = ['view', 'create_workflow', 'use_templates'];
  } else if (member.role === 'teacher') {
    req.permissions = ['view', 'edit_course', 'publish_template', 'view_reports'];
  }
  
  next();
}
